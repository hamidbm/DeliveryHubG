# Next: Watchers/Subscriptions v1 (Bundles + Milestones) ## Goal Allow users to subscribe to bundles and milestones to receive relevant notifications and appear as stakeholders in governance workflows. This builds on existing notifications preferences and pipeline. --- ## Part A — Data Model Add collection: `notification_watchers` Schema: ```ts { userId: string, scopeType: 'BUNDLE' | 'MILESTONE', scopeId: string, // bundleId or milestoneId createdAt: string, createdBy: string } ``` Indexes: - unique: { userId: 1, scopeType: 1, scopeId: 1 } - lookup: { scopeType: 1, scopeId: 1 } --- ## Part B — APIs Add endpoints: 1) Watch/unwatch - POST /api/watchers body: { scopeType, scopeId } - DELETE /api/watchers body: { scopeType, scopeId } 2) List my watchers - GET /api/watchers?scopeType=... (optional) 3) List watchers for a scope (admin/owner only) - GET /api/watchers/scope?scopeType=BUNDLE&scopeId=... RBAC: - any authenticated user can watch/unwatch for themselves - only Admin/CMO or bundle owner can list watchers for a bundle - only Admin/CMO can list watchers for milestones (or define milestone ownership rule) --- ## Part C — Notification Routing Update Update `src/services/notifications.ts` recipient resolution: For each event/notification type: - determine impacted bundleIds and milestoneIds from payload: - milestone events -> milestoneId and bundles in scope (derive from work items if needed; keep best-effort) - work item events -> bundleId and milestoneIds on item - cross-bundle dependency event -> both bundleIds and both milestoneIds if present Then add recipients: - watchers of those bundleIds - watchers of those milestoneIds Respect: - global enabledTypes - user mutedTypes - digest opt-in rules Avoid duplicates. --- ## Part D — UI Integration Add Watch buttons in key places: 1) Bundle Profile (Applications module) - “Watch bundle” toggle in Delivery Intelligence section - Shows current state for the user 2) Milestone Planning view - “Watch milestone” toggle next to milestone header 3) Program page (optional) - show “Watching” indicator and a link to manage watchers Minimal UI: a toggle button + confirmation toast/message. --- ## Part E — Tests Add script: scripts/test-watchers.ts Test: - user can watch/unwatch bundle - duplicates prevented - routing: - when cross-bundle BLOCKS created, watchers of affected bundles receive notification - when milestone status changes, milestone watchers receive notification Add npm script: test:watchers Include in test:api runner. --- ## Deliverables - notification_watchers collection + indexes - watchers APIs + RBAC - updated notification routing to include watchers - watch toggles in Bundle Profile + Milestone Planning - tests + docs update (brief)
