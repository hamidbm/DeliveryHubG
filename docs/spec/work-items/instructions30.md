# Next: Critical Path v1 (Dependency-Aware Guidance) ## Goal Compute and surface a milestone’s critical path and “top actions” so the system can answer: - What chain of dependencies is driving the milestone ETA? - Which blockers should be resolved first to reduce slip risk? - Which work items are critical vs non-critical? This should be explainable, fast, and based on existing dependency model (BLOCKS edges) + estimates. --- ## Part A — Critical Path Computation (Server-side) Add service: - src/services/criticalPath.ts Implement: computeMilestoneCriticalPath(milestoneId, options?) Inputs: - work items scoped to milestone - BLOCKS edges among those items - estimates: - primary: storyPoints - fallback: timeEstimateHours (convert to points using a constant, e.g. 1pt=4h) OR skip if missing - item status: - DONE items contribute 0 remaining - open items contribute remaining estimate Algorithm (v1, deterministic DAG-only): 1. Build directed graph from BLOCKS edges (blocker -> blocked). 2. Restrict to items in the milestone (ignore external nodes for now, but record external blockers separately). 3. If cycles exist, return a partial result: - cycleDetected=true, include involved nodes list 4. Compute longest path by remaining estimate: - topo sort - dp[node] = remaining(node) + max(dp[pred]) where pred are blockers 5. Determine: - criticalPathNodes (nodes on the max path ending at the node with max dp) - criticalPathRemainingPoints - criticality score per node: - critical if in path - near-critical if within 10% of max dp Return shape: ```ts type CriticalPathResult = { milestoneId: string; cycleDetected: boolean; cycleNodes?: Array<{ id; key; title }>; criticalPath: { nodes: Array<{ id; key; title; status; remainingPoints; sprintId? }>; remainingPoints: number; }; nearCritical: Array<{ id; key; title; slackPoints: number }>; externalBlockers: Array<{ blockerId; blockerKey; blockerTitle; blockedId; blockedKey; blockedTitle; blockerMilestoneId?: string; blockerBundleId?: string; }>; topActions: Array<{ type: 'UNBLOCK'|'ESTIMATE'|'ASSIGN'|'SCOPE_REDUCE'; itemId: string; key: string; title: string; reason: string; }>; }; ``` Top actions (simple heuristics): - UNBLOCK: items on critical path that are currently blocked (derived isBlocked) - ESTIMATE: critical/near-critical items missing storyPoints - ASSIGN: critical items missing assignee - SCOPE_REDUCE: critical path remaining exceeds capacity remaining by threshold (optional) --- ## Part B — API Endpoints Add: - GET /api/milestones/[id]/critical-path Params: - includeExternal=true|false (default true) - limit= (default 50 nodes) Return CriticalPathResult. Add caching: - in-memory cache keyed by milestoneId for ~30s is fine (server-side) - invalidate if scope changes, dependency changes, or status changes (best-effort; v1 can accept short TTL only) --- ## Part C — UI Integration ### C1.
