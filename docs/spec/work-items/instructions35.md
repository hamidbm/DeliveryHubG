# Next: Change Feed v1 (Delivery-facing activity timeline) ## Goal Add a unified, readable change feed for: - Milestones - Bundles (Bundle Profile) - Program (global) This should transform low-level events into a human timeline with deep links. No new event generation required; consume existing events. --- ## Part A — Feed API Add endpoint: - GET /api/feed Query params: - scopeType = 'PROGRAM' | 'BUNDLE' | 'MILESTONE' - scopeId (required for BUNDLE/MILESTONE) - limit (default 50) - cursor (for pagination) Behavior: - Fetch from `events` collection (reuse existing admin events query logic but scoped). - Filter event types relevant to delivery: - milestones.* - sprints.* - dependency.* - criticalpath.* - milestones.scope.* - notifications.* (optional, maybe too noisy) - policy.* (exclude by default) - perf.* (exclude by default) - Map each event into a `FeedItem` with: ```ts type FeedItem = { id: string; occurredAt: string; actor: { userId?: string; email?: string; name?: string }; title: string; summary: string; severity: 'info'|'warn'|'critical'; links: Array<{ label: string; href: string }>; rawType: string; raw?: any; // only when includeRaw=true (default false) }; ``` Transform examples: - milestones.milestone.statuschanged -> "Milestone started" / "Milestone marked done" - milestones.scope.approved -> "Scope change approved: +5 items" - dependency.criticalpath.escalation -> "Escalated external blocker ABC-123" - criticalpath.action.executed -> "Requested estimate for XYZ-9" - sprints.sprint.statuschanged -> "Sprint closed" Pagination: - cursor should be based on occurredAt + _id (stable). --- ## Part B — UI Surfaces ### B1. Bundle Profile In Delivery Intelligence section add "Recent activity" (top 10) with "View all" link: - scopeType=BUNDLE&scopeId=... ### B3. Program page Add a panel "Program activity" (top 20): - scopeType=PROGRAM Each feed item: - shows time, actor, title, summary - includes deep links (milestone/work item/program) --- ## Part C — Filtering and UX - Provide filter toggles: - Governance (milestone/sprint) - Scope changes - Dependencies/Critical path - Integrations (optional) Default filters: governance + scope + dependency. - Provide "Load more" pagination. --- ## Part D — RBAC - Feed is delivery-facing, so: - any authenticated user can view PROGRAM feed (unless you want to restrict) - BUNDLE feed visible to: - Admin/CMO - bundle owners - bundle watchers (optional) - MILESTONE feed visible to: - Admin/CMO - bundle owners/watchers for bundles in milestone scope (best effort) If too complex in v1: allow all authenticated users; document that feed respects existing data visibility later. --- ## Part E — Tests Add: - scripts/test-feed.ts Seed events and verify: - correct scoping filter - correct transformations for key types - pagination works Add npm script: test:feed and include in test:api. --- ## Deliverables - /api/feed endpoint with transformations + pagination - milestone, bundle, program UI panels - filters + load more - tests + docs update
