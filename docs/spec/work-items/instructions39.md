# Next: GitHub Integration v1 (One-way enrichment for PR/commit activity) ## Goal Link GitHub PRs/commits to DeliveryHub work items and surface activity in: - Work item details panel - Change Feed - Critical Path panel (signal: active/stale) - Optional: Roadmap cards (small activity badge) Keep it v1: - one-way ingest only - manual sync endpoint (no scheduler yet) - minimal mapping rules - no writeback to GitHub --- ## Part A — Data Model Extend WorkItem: ```ts github?: { repo?: string; // owner/repo prs?: Array<{ number: number; title: string; url: string; state: 'open'|'closed'|'merged'; updatedAt: string; author?: string; }>; lastSyncedAt?: string; } ``` Add collection (optional but better): - `github_links` ```ts { workItemId: string; repo: string; prNumber: number; url: string; state: string; title: string; updatedAt: string; } ``` Indexes: - github_links: { repo: 1, prNumber: 1 } unique - github_links: { workItemId: 1 } If you store only inside WorkItem, add no new collection; but a link table scales better. --- ## Part B — Mapping Rules (v1) Detect work item key references in PR title/body/branch: - pattern: `[A-Z]+-\d+` (Jira style) - also allow `WI-\d+` if you have your own key style Mapping approach: - when syncing a repo, scan recent PRs and link to work items whose key matches --- ## Part C — GitHub Client Create: - src/services/github.ts Auth: - env vars: - GITHUB_TOKEN (PAT) - GITHUB_REPOS (comma-separated owner/repo) Use GitHub REST API. Support mock responses for tests. Fetch: - list PRs updated in last N days (default 14) - include title/body/state/updatedAt/url --- ## Part D — Admin Sync API Add endpoints (Admin/CMO only): - GET /api/admin/integrations/github/preview?repo=...&limit=... - POST /api/admin/integrations/github/sync body: { repo?: string, sinceDays?: number, // default 14 limit?: number, mode?: 'UPSERT' } Behavior: - fetch PRs - parse work item keys - upsert github_links (or embed in work items) - emit event: - integrations.github.sync.completed with counts --- ## Part E — UI Surfaces 1) Admin → Integrations → GitHub - config status - preview - manual sync - last run summary 2) Work item details panel - show linked PRs (title, state, updatedAt, link) 3) Change Feed - show new events: - workitem.github.pr.linked (when a new link appears) - workitem.github.pr.merged (when state changes to merged) Only emit these events when link is new or state transitions. 4) Critical Path panel (signal chip) - for each critical item: - if linked open PR updated in last 3 days: “Active” - if no PRs or stale: “Stale” This is a hint, not a gate. --- ## Part F — Tests Add: - scripts/test-github-integration.ts Mock GitHub responses: - PR title references a work item key -> link created - link deduped - state change emits event - work item details includes PR list Add npm script and include in test:api. --- ## Deliverables - github service - admin preview/sync APIs + admin integrations UI - work item details PR section - change feed mapping for github-linked events - critical path activity hints - tests + docs update
