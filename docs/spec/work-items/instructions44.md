# Next: Backup/Restore + Environment Promotion v1 (Operational Safety) ## Goal Add a safe, admin-only backup/export and restore/import mechanism for critical collections: - policies (global + overrides) - notification settings + prefs (optional) - bundles + assignments - milestones - scope change requests - (optionally) work items (or work item metadata only) Provide: - export endpoint that returns a versioned JSON bundle - import endpoint that validates and performs a dry-run diff - restore protections (idempotency, required confirmations, audit events) - admin UI to run export/import Keep v1 bounded to configuration + planning metadata; avoid huge work item exports unless requested. --- ## Part A — Define Backup Bundle Format Create a versioned export format: ```ts type BackupBundleV1 = { version: 1; exportedAt: string; exportedBy: string; environment?: string; // optional label collections: { delivery_policies?: any[]; delivery_policy_overrides?: any[]; notification_settings?: any[]; notification_user_prefs?: any[]; bundles?: any[]; bundle_assignments?: any[]; milestones?: any[]; scope_change_requests?: any[]; }; counts: Record<string, number>; }; ``` Include a checksum hash (optional) for integrity. --- ## Part B — Export API (Admin/CMO only) Add: - GET /api/admin/backup/export?include=policies,overrides,bundles,milestones,... Security: - Admin/CMO only - optionally require X-Admin-Export-Secret env var header for extra safety Behavior: - exports selected collections - strips secrets (e.g., Jira tokens) if any stored - emits event: admin.backup.exported with counts --- ## Part C — Import API with Dry Run Add: - POST /api/admin/backup/import Body: ```ts { bundle: BackupBundleV1, mode: 'DRY_RUN' | 'APPLY', options: { allowUpsert: boolean; // default true overwritePolicies: boolean; // default false overwriteOverrides: boolean; // default false }, confirmation?: { phrase: string; // required for APPLY, e.g. "IMPORT_BACKUP" } } ``` Dry-run response should include: - collections affected - upserts vs creates counts - any validation errors - diff summary for policies/overrides Apply mode: - performs upserts by stable ids (_id / bundleId / milestoneId) - writes audit events: - admin.backup.import.started / completed / failed - returns summary --- ## Part D — Admin UI Add Admin → Operations → Backup/Restore page: - Export section: - checkboxes for collections - download JSON (or copy to clipboard) - Import section: - paste JSON or upload file - run Dry Run - shows diff + counts - confirm phrase + Apply UI should be explicit and safe. --- ## Part E — Tests Add scripts/test-backup.ts: - seed a few docs across collections - export bundle includes expected docs + counts - dry run detects changes - apply updates docs (upsert) and emits events - non-admin blocked Add npm script: - test:backup Include in test:api. --- ## Part F — Docs Update docs/wiki/Operations.md: - how to export/import - recommended cadence - how to promote configs between environments --- ## Deliverables - export/import endpoints - dry-run diff and apply safeguards - admin UI - tests + docs
