# Next: Commitment Review v1 (COMMIT gating using Monte Carlo + capacity + quality)
## Goal Introduce a formal "Commitment Review" step that gates moving a milestone to COMMITTED. The review should evaluate: - Monte Carlo hit probability and P80 vs endDate - capacity plan overcommit signal for the owning bundle(s) - data quality score - critical path external blockers count - staleness critical count - risks high/critical Result: - PASS or FAIL with reasons - optional Admin/CMO override with reason - persist a snapshot so audits show "why we committed" Keep v1 minimal: milestone-level review only.

--- ## Part A — Review computation service Add: - src/services/commitmentReview.ts Function: - evaluateMilestoneCommitReview(milestoneId) -> { canCommit, score, band, checks[], snapshot } Checks (v1): 1) Forecast confidence: - if monteCarlo.enabled and hitProbability < policy.commitReview.minHitProbability => fail - if P80 > endDate => fail (or warn based on policy) 2) Capacity credibility: - if bundle capacity plan shows overcommit within milestone window by > threshold => fail 3) Data quality: - if dataQuality.score < policy.readiness.milestone.blockScoreBelow (or dedicated threshold) => fail 4) Dependency risk: - if critical path external blockers > 0 and policy.commitReview.blockOnExternalBlockers => fail/warn 5) Staleness: - if criticalStaleCount >= policy.commitReview.maxCriticalStale => fail/warn 6) Risk: - if high/critical risks exceed threshold => fail/warn Return: ```ts type CommitmentReview = { milestoneId: string; canCommit: boolean; score: number; // 0..100 band: 'GREEN'|'YELLOW'|'RED'; checks: Array<{ key: string; status: 'PASS'|'WARN'|'FAIL'; detail: string }>; snapshot: { rollup: MilestoneRollup; // include policy refs monteCarlo?: MonteCarloForecast; capacitySignal?: { overcommitMax: number; horizonWeeks: number }; criticalPath?: { externalCount: number; remainingPoints: number }; staleness?: { criticalStaleCount: number }; }; }; ```
