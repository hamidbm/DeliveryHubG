No planning engine logic should be modified. The roadmap will display: - milestone timelines - probabilistic forecast windows - dependency flows - capacity utilization - risk signals - environment phases All existing intelligence from earlier phases must be reused. --- # Current Limitations The current timeline: - shows only basic milestone bars - has minimal visual differentiation - lacks dependency visibility - lacks milestone grouping - does not show environment phases - does not visually express uncertainty - wastes horizontal space - does not allow zoom levels --- # Target Capabilities The roadmap must support: 1. Zoomable timeline 2. Forecast confidence visualization 3. Dependency arrows 4. Environment phase overlays 5. Capacity heat indicators 6. Risk badges 7. Milestone grouping 8. Portfolio layering 9. Intelligent tooltips 10. Interactive navigation --- # 1 Zoomable Timeline Add timeline zoom levels. Supported levels: ``` Quarter Month Sprint Week ``` Controls: ``` [ - ] Zoom Out [ + ] Zoom In ``` Zoom must update: - grid resolution - milestone bar precision - label density --- # 2 Milestone Bar Design Replace the current single bar with a layered structure. Milestone visualization should include: ``` | forecast band | | committed bar | ``` ### Committed timeline Solid bar. Represents: ``` plannedStart → plannedEnd ``` ### Forecast band Transparent overlay. Represents: ``` P50 → P90 probabilistic forecast ``` Example visual: ``` |===== committed =====| |---- forecast band -------| ``` --- # 3 Forecast Confidence Indicators Add confidence indicators directly on milestone bars. Confidence levels: ``` High Medium Low ``` Visual encoding: ``` Green dot → High Amber dot → Medium Red dot → Low ``` Tooltip must show: ``` Confidence: Medium On-time probability: 68% P50 date: Mar 28 P90 date: Mar 30 ``` Data source: ``` probabilisticForecastingEngine ``` --- # 4 Dependency Visualization Show milestone dependencies with arrows. Dependencies come from: ``` BLOCKS links ``` ### Behavior When a milestone depends on another: ``` Milestone A → Milestone B ``` Render curved arrows connecting bars. Color coding: ``` Gray → normal Red → blocking delayed milestone Amber → dependency risk ``` Hovering arrow should display: ``` Dependency: M1 → M2 Blocked items: 4 Dependency pressure: High ``` --- # 5 Environment Phase Overlay Overlay environment phases across the timeline. Source: ``` application_planning_metadata.environments[] ``` Example: ``` DEV SIT INT UAT PROD ``` Each environment appears as a **horizontal band behind milestones**. Example: ``` | DEV | SIT | INT | UAT | PROD | ``` Milestones visually align with the phases they occur in. This helps answer: ``` Which milestones happen during UAT? Which milestones reach PROD? ``` --- # 6 Capacity Heat Overlay Milestone bars should include a **capacity intensity indicator**. Based on: ``` targetCapacity vs actual load ``` Color gradient: ``` Green → healthy Amber → nearing capacity Red → overloaded ``` Displayed as: ``` subtle glow behind milestone bar ``` Tooltip: ``` Target capacity: 120 points Allocated: 108 points Utilization: 90% ``` --- # 7 Risk Visualization Risk signals must appear directly on the timeline. Hover tooltip shows details. --- # 8 Milestone Grouping Allow milestones to be grouped by: ``` Application Bundle Theme Owner ``` Add grouping dropdown: ``` Group by: [ None ] [ Application ] [ Owner ] [ Theme ] ``` Grouped milestones appear as collapsible sections. --- # 9 Portfolio Overlay When viewing a bundle or portfolio: Allow roadmap to show **multiple plans stacked**.
