# Architecture Module

The Architecture module is the LeanIX-like design and governance layer of DeliveryHub.

It manages architecture diagrams, design artifacts, structured review cycles, and the connection between technical decisions and delivery execution.

## What This Module Is For

Large programs need more than a task tracker. They also need a place where architecture decisions are created, reviewed, and tied back to real delivery outcomes.

This module provides that capability.

## Core Capabilities

### Diagram gallery

Teams can manage a catalog of architecture diagrams and related assets in one place instead of scattering them across local files and slide decks.

### Structured diagram creation

The module supports a guided creation flow so new diagrams can begin from:

- a blank canvas
- a reusable template
- an uploaded artifact

This encourages more consistent architecture documentation across teams.

### Multiple diagram styles

The module supports a practical range of architecture formats, including:

- Mermaid
- mind map style diagrams
- Draw.io and uploaded architecture files

### Business and delivery metadata

Architecture artifacts can be connected to:

- bundle
- application
- milestone
- document type

This is important because it allows architecture to participate in delivery governance instead of remaining detached from the program.

### Architecture reviews

Diagrams can be submitted into a structured review workflow with assigned reviewers, review comments, review history, and visible completion flow.

### Review-linked work items

Each review request creates visible follow-up work in the Work Items module so architecture governance is not hidden.

### Comments and activity trail

Architecture collaboration is traceable through comments and activity history, making it easier to understand what changed and why.

## Typical Ways Teams Use It

### For architects

Create diagrams, standardize them through templates, and run formal reviews when design governance matters.

### For PMO and delivery leadership

Use architecture review visibility to understand whether technical governance is enabling delivery, delaying it, or uncovering rework.

### For upper management

Use this module as evidence that critical design work is being governed in a visible and disciplined way.

## How This Module Connects to the Rest of DeliveryHub

- **Applications** supplies the bundle and application context around architecture work.
- **Work Items** tracks review-related execution follow-up.
- **Reviews** provides the shared review cycle model.
- **Wiki** can embed or reference architecture content in broader documentation.

## Why This Module Matters

Architecture often becomes a disconnected side process in transformation programs.

DeliveryHub avoids that by treating architecture as part of the operational system, with visible governance, history, and links to execution.
