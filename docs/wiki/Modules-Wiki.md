# Wiki Module

The Wiki module is the knowledge and documentation management layer of DeliveryHub.

It provides the Confluence-like side of the platform while staying tightly connected to review workflows, execution, and AI-assisted reading.

## What This Module Is For

Program delivery depends on more than tasks. Teams also need a durable place for:

- authored pages
- uploaded working documents
- review packages
- design notes
- operational references
- supporting spreadsheets and artifacts

The Wiki module is where that knowledge lives.

## Core Capabilities

### Authored pages and uploaded assets

The module supports both authored markdown pages and uploaded files such as Word documents, PDFs, images, and Excel files.

This makes it useful for both structured documentation and real-world program artifacts.

### Navigation for large document sets

The Wiki includes tree-based navigation, breadcrumbs, and filters so users can work with large collections of documentation without getting lost.

### Word-to-markdown knowledge capture

Word files can be brought into the system and converted into editable markdown-based content while preserving the original file for download.

This is particularly useful for migration programs where many key documents start in Office formats.

### Excel ingestion and preview

Excel files can be previewed in table or tile style views, making spreadsheet-based content easier to explore inside the product.

### Rich markdown support

The Wiki supports modern documentation patterns such as:

- formatted markdown
- tables
- code formatting
- Mermaid diagrams
- mind maps

### Formal review workflows

Pages and uploaded artifacts can be submitted into a formal review cycle with reviewer feedback, cycle history, and linked follow-up work.

### AI assistance for reading and synthesis

The module supports AI actions such as:

- summarization
- key decision extraction
- assumptions extraction
- Q&A against content
- template assistance for new pages

## Typical Ways Teams Use It

### For delivery teams and PMs

Use the Wiki as the controlled home for important program artifacts instead of leaving them spread across folders and email chains.

### For architects and reviewers

Use it to publish design material, supporting evidence, and reviewable documentation tied to bundles and applications.

### For upper management

Use it to find the latest program documentation and consume long artifacts faster with summaries and guided navigation.

## How This Module Connects to the Rest of DeliveryHub

- **Reviews** handles the shared review cycle model for reviewable documents.
- **Work Items** tracks the visible follow-up created by review requests.
- **Architecture** can complement the Wiki with formal diagram assets.
- **AI Insights** and Wiki AI actions help users understand long-form content more efficiently.

## Why This Module Matters

Documentation loses value when it is disconnected from execution.

The Wiki module keeps documents, comments, reviews, and AI-assisted reading connected to the rest of DeliveryHub so knowledge remains actionable.
