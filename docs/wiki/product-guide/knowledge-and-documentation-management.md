# Knowledge and Documentation Management

This capability is the **Knowledge** layer of DeliveryHub, powered by the **Wiki** module.

## What It Means

This is the Confluence-like side of the product. It is where teams create living documentation, upload supporting files, organize structured knowledge, and connect documents to delivery work and review workflows.

## What Is Implemented

### Authored pages and uploaded assets

The module supports:

- markdown pages
- Word documents
- PDFs
- images
- Excel files

This makes it useful for both structured authored content and real-world program artifacts.

### Navigation built for large documentation sets

Users can browse content through:

- tree navigation
- breadcrumbs
- filters by business context such as space, bundle, application, and document type

### Word-to-markdown knowledge capture

Word documents can be brought into the knowledge system and turned into editable markdown-based content while preserving the original file for download.

This is valuable for migration programs because many important documents already exist in Office formats and need to become easier to maintain collaboratively.

### Excel ingestion and preview

Excel files can be previewed in business-friendly ways such as tile and table modes, which makes the module useful for more than static documents.

### Markdown with rich embedded content

Knowledge pages support:

- formatted markdown
- tables
- syntax highlighting
- Mermaid diagrams
- mind maps

This allows teams to combine narrative content, structured tables, and visual explanation in one place.

### Knowledge-linked review workflow

Documents and assets can be submitted for formal review, with review cycles and linked work items so governance becomes part of visible delivery.

### Comments, activity, and traceable collaboration

The Knowledge area also supports comments and activity tracking so discussions stay attached to the artifact instead of being lost in email or chat.

### AI assistance for knowledge work

The module supports AI-driven actions such as:

- summarize content
- extract key decisions
- extract assumptions
- answer questions against pages and assets
- help generate templates for new pages

These capabilities are especially useful for long program documents that many stakeholders need to consume quickly.

## Why These Features Matter

- program knowledge becomes durable and reusable
- documentation stays connected to delivery and governance
- large document sets become easier to navigate and explain
- teams spend less time re-reading long artifacts because AI can highlight key points

## Who Gets the Most Value

### Delivery teams and PMs

Use the Knowledge module to keep program documents tied to the work they drive.

### Architects and reviewers

Use it to publish design material and move that material into formal feedback cycles.

### Leadership

Use it as a reliable place to find current program documentation without relying on scattered files and email chains.

## How Executives Should Use This in Practice

### CIO

Use the Knowledge area to find current migration documents, review packages, and program decisions without depending on slide decks or inbox forwarding.

### VP

Use it to quickly orient yourself on a bundle or application before a review meeting by reading the latest documents and AI-generated summaries.

### PMO

Use the wiki as the controlled home for key program artifacts so documentation, comments, and review cycles stay linked to execution.

### Architects

Publish technical material here, keep it updated as a living document, and route important artifacts into formal review when needed.

### Delivery managers

Use pages, uploaded assets, comments, and AI summaries to reduce time spent searching through long documents during planning and issue resolution.
