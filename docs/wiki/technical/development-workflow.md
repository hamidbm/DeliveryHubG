# Development Workflow

This page is the technical handbook for contributing safely to DeliveryHub.

## Core Working Rules

- keep MongoDB access server-side
- prefer stable API shapes before UI expansion
- avoid breaking markdown rendering and sanitization
- use the documented commands for build, lint, bootstrap, and tests

## Common Commands

```bash
npm run dev
npm run build
npm run start
npm run lint
npm run lint:ci
npm run test:api
```
